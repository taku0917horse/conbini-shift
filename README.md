# コンビニシフト管理

コンビニ・小規模店舗向けのシフト管理 Web アプリです。  
ブラウザだけで動作し、インストール不要でスマートフォンのホーム画面に追加できます（PWA）。

## 画面構成

画面下部に 4 つのタブがあります。

| タブ | 内容 |
|------|------|
| 📅 シフト | 曜日ごとのシフト表（バーチャート） |
| 👥 従業員 | 従業員の一覧・追加・編集 |
| ⚙️ 必要人数 | 不足リスト／募集中リスト／ルール設定 |
| 📤 共有・データ | シフト表の画像保存・JSON エクスポート／インポート |

## シフトを管理する

### シフト表の見方

- **シフトタブ**を開くと、曜日タブ（月〜日）で切り替えられるバーチャートが表示されます。
- 縦軸は時間（3:00〜翌6:00）、各バーが 1 件の勤務を表します。
- バーをタップすると勤務の編集画面が開きます。
- **赤いオーバーレイ**が表示されている時間帯は必要人数が不足しています。

### 表示サイズの切り替え

ヘッダー右の **「コンパクト」「標準」「ゆったり」** ボタンをタップするたびに、時間軸の高さが切り替わります。

### 勤務を追加する

1. ヘッダー右の **「＋勤務追加」** をタップします。
2. 以下を設定して **「保存」** をタップします。

| 項目 | 説明 |
|------|------|
| 従業員 | プルダウンで選択 |
| 曜日 | 複数選択可。同じ内容を複数曜日にまとめて登録できます |
| プリセット | よく使う時間帯ボタン（3-6 / 6-9 / 9-13 / 13-17 / 17-22 / 22-3）をタップすると開始・終了が自動入力されます |
| 開始時刻 / 終了時刻 | 直接入力も可能。翌日にまたがる場合（例: 22:00〜翌6:00）は終了に `06:00` と入力します |
| 休憩（分） | 「なし / 45分 / 60分」ボタン、または数値で直接入力 |
| 仮（募集中）の時間範囲 | **店長関連の従業員のみ表示**。詳細は[下記](#仮募集中シフト) |

### 勤務を編集・削除する

バーをタップすると編集画面が開きます。内容を変更して **「保存」**、または **「削除」** をタップします。

## 従業員を管理する

### 従業員リストの見方

- 従業員は**区分（明朝 / 早朝 / 日勤 / 夕勤 / 夜勤）ごと**にグループ表示されます。
- タップすると週間スケジュール（各曜日の勤務時間・週合計実働）が確認できます。

### 従業員を追加・編集する

**「＋追加」** または従業員名をタップして編集画面を開きます。

| 項目 | 説明 |
|------|------|
| 名前 | シフト表・リストに表示される氏名 |
| 略称 | 印刷・画像用（2〜4 文字）。未入力の場合は名前の先頭 2 文字が使われます |
| 区分 | 主な勤務帯（明朝 / 早朝 / 日勤 / 夕勤 / 夜勤）。リストでのグループ分けに使います |
| 色 | バーの色。8 色から選択 |
| メモ | 覚え書き（シフト判定には影響しません） |
| 店長関連 | ON にすると**仮（募集中）シフト**を設定できるようになります |

### 店長関連フラグを外すと

その従業員の全勤務から「仮」の設定が自動的に解除されます。

## 必要人数を設定する

**必要人数タブ → ルール設定** で、時間帯ごとに最低必要人数を設定できます。

### ルールの追加・編集

**「＋追加」** をタップして曜日・時間・人数を設定します。

| 項目 | 説明 |
|------|------|
| 曜日 | ルールを適用する曜日 |
| 時間 | 開始〜終了（プリセット使用可） |
| 必要人数 | **－ / ＋** ボタンで設定 |

- 時間帯が重複するルールがある場合は**最大値**が適用されます。
- **「全日コピー」**: 選択した曜日のルールを残り全曜日にコピーします（既存ルールは上書き）。

---

## 不足リスト

**必要人数タブ → 不足リスト** で、必要人数を満たせていない時間帯を一覧表示します。

- 時間帯フィルタで絞り込めます（明朝 3–6 / 早朝 6–9 / 日勤 9–17 / 夕勤 17–22 / 夜勤 22–翌3）。
- 行をタップすると、その時間帯で勤務追加モーダルが開きます（日時が自動入力されます）。

---

## 募集中リスト

**必要人数タブ → 募集中** で、仮（募集中）として登録されている勤務を一覧表示します。

### 仮（募集中）シフト

本来は店長が穴埋めで入る予定だが、正規のスタッフを募集中の状態を管理できます。

- **設定できるのは店長関連フラグが ON の従業員のみ**です。
- 勤務全体ではなく、**一部の時間帯だけを仮**にすることができます。  
  例: 9:00〜22:00 の勤務のうち 17:00〜22:00 だけを募集中にする。

#### シフトチャートでの表示

仮の時間範囲はバーの中に**斜線パターン**で表示されます。バー自体は 1 本のままです。

#### 募集中リストでの表示

- 仮の時間範囲（例: 17:00〜22:00）と担当者名が表示されます。
- 時間帯フィルタで絞り込めます。複数時間帯にまたがる仮シフトは、各フィルタで**その時間帯に重なる部分だけ**表示されます。

#### 人数カウントについて

仮シフトも**必要人数のカウントに含まれます**（人手は足りている扱い）。不足リストには反映されません。

## データの共有・バックアップ

**共有・データタブ**から操作できます。

### 画像保存

**「画像保存」** をタップすると、7 曜日分のシフト表が PNG 画像として生成されます。

- **スマートフォン**: Web 共有シート（LINE・メール等への送信）が開きます。
- **PC / 非対応ブラウザ**: 画像が新しいタブで開くので、長押し（モバイル）またはダウンロードで保存できます。

### エクスポート（書き出し）

**「書き出し」** をタップすると、従業員・シフト・ルールの全データが JSON ファイルとしてダウンロードされます。他の端末やスタッフへの共有に使えます。

### インポート（読み込み）

**「読み込み」** をタップして JSON ファイルを選択します。

- 読み込む前に**現在のデータが上書きされる**旨の確認ダイアログが表示されます（日時も表示）。
- エクスポートしたファイルのみ読み込めます（version: 1）。

## ホーム画面への追加

このアプリは PWA（Progressive Web App）に対応しています。ブラウザからホーム画面に追加すると、アプリのように起動できます。

**iOS（Safari）**: 共有ボタン → 「ホーム画面に追加」  
**Android（Chrome）**: メニュー → 「ホーム画面に追加」

---

## 時刻の扱い（深夜またぎ）

このアプリは **3:00 を 1 日の起点**として扱います（例: 夜勤 22:00〜翌3:00 は同じ曜日の勤務として管理）。

- 深夜 3:00 を越える終了時刻は **「翌 HH:MM」** と表示されます。
- 勤務・ルールの入力で、翌日にまたがる場合は終了時刻に実際の時刻をそのまま入力します（例: 翌6:00 → `06:00`）。終了が開始より小さい場合、自動的に翌日として処理されます。
- 前日から続く深夜勤務は、翌曜日のシフト表にも自動的に表示されます（両日表示）。
